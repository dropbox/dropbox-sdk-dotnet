Parameters

name
Type: SystemString
The last component of the path (including extension). This never contains a slash.
id
Type: SystemString
A unique identifier for the file.
clientModified
Type: SystemDateTime
For files, this is the modification time set by the desktop client when the file was added to Dropbox. Since this time is not verified (the Dropbox server stores whatever the desktop client sends up), this should only be used for display purposes (such as sorting) and not, for example, to determine if a file has changed or not.
serverModified
Type: SystemDateTime
The last time the file was modified on Dropbox.
rev
Type: SystemString
A unique identifier for the current revision of a file. This field is the same rev as elsewhere in the API and can be used to detect changes and avoid conflicts.
size
Type: SystemUInt64
The file size in bytes.
pathLower (Optional)
Type: SystemString
The lowercased full path in the user's Dropbox. This always starts with a slash. This field will be null if the file or folder is not mounted.
pathDisplay (Optional)
Type: SystemString
The cased path to be used for display purposes only. In rare instances the casing will not correctly match the user's filesystem, but this behavior will match the path provided in the Core API v1. Changes to the casing of paths won't be returned by ListFolderContinueAsync(ListFolderContinueArg). This field will be null if the file or folder is not mounted.
parentSharedFolderId (Optional)
Type: SystemString
Deprecated. Please use ParentSharedFolderId or ParentSharedFolderId instead.
mediaInfo (Optional)
Type: Dropbox.Api.FilesMediaInfo
Additional information if the file is a photo or video.
sharingInfo (Optional)
Type: Dropbox.Api.FilesFileSharingInfo
Set if this file is contained in a shared folder.
propertyGroups (Optional)
Type: System.Collections.GenericIEnumerablePropertyGroup
Additional information if the file has custom properties with the property template specified.
hasExplicitSharedMembers (Optional)
Type: SystemNullableBoolean
This flag will only be present if include_has_explicit_shared_members is true in ListFolderAsync(ListFolderArg) or GetMetadataAsync(GetMetadataArg). If this flag is present, it will be true if this file has any explicit shared members. This is different from sharing_info in that this could be true in the case where a file has explicit members but is not contained within a shared folder.
